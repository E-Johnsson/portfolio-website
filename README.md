# Apparel Design Portfolio (V1) — Stepping Stone Project

**Status:** Shelved. Kept as a record of what I learned, not as an active build.

## What it was

A side-scrolling portfolio site meant to showcase the design side of an independent apparel project — graphic design, marketing campaigns, photoshoots, and finished products — as a single polished, browsable experience rather than a static grid of images.

## Tech stack

- **Next.js** (TypeScript), bootstrapped with `create-next-app`
- **Geist** font family (via `next/font`)
- Intended deployment target: **Vercel**

## Why it's shelved

This was my first project working with an AI coding workflow (Claude Code + VS Code), and I built it before I understood how to scope a project or direct that workflow effectively. Two things caught up with it:

1. **Scope was too big for a first project.** I split it into six separate pages, each with its own design focus. It ended up reading more like a design-school portfolio exercise than a tight, purposeful site — there was more structure than the content actually needed.
2. **AI-assisted layout wasn't the right tool for the job.** The site ran correctly (it's fully functional locally), but getting images placed in a genuinely aesthetic, intentional way — the actual point of a design portfolio — wasn't something the workflow I was using handled well. Visual layout turned out to need a much more hands-on, iterative process than I was giving it.

## What I took from it

This project is what pushed me toward a more deliberate way of working with AI-assisted development: writing a full spec before touching code, breaking builds into small single-concept slices, and understanding every piece before moving to the next one. That approach is what shaped my next project, [an AI-powered personal organizer](#) — built slice by slice, with far tighter scope control from the start.

## Status of the code

The code is complete and still exists locally; it runs correctly on `localhost:3000` via the standard Next.js dev flow. It hasn't been deployed or pushed further, since the layout issues made it not worth polishing further in its current form. It's kept here as a milestone, not a maintained project.
